# Career JD Matcher (v1.2)

경력직 이직 지원자가 채용공고에 맞춰 지원 패키지를 정교화하는 통합 스킬이다. **5단계를 한 번의 호출로 끝낸다.**

## v1.2 변경 요약

- **분석 보고서 docx 신규 추가**: 채팅 마크다운 Section 1~6 + 면접 예상 질문을 통합한 별도 docx 파일을 항상 생성. 최종 산출물이 **2개 → 3개**로 확장 (이력서 + 자소서 + 분석보고서).

## v1.1 변경 요약 (유지)

- **정체성 확인 게이트 (Step 4-0)**: 자소서 톤 결정 전 직무 정체성을 1회 명시적으로 확인.
- **Fact-Checker 페르소나 (Stage 0)**: 리라이팅 사이클 종료 시 원본 대조 Pass/Fail 게이트.
- **의사결정 매트릭스 (Section 6)**: 지원 권장도를 Growth/Safety/Meaning/Fit 4축 100점 가중평균으로 산출.

## 핵심 원칙 (스킬 전체에 적용)

### 1. 거짓 사실 절대 금지 (최우선)

사용자 경력기술서·자소서에 없는 경험을 **어떤 이유로도** 추가하지 않는다. JD 매칭률을 높이려고, 페르소나 점수를 끌어올리려고, 또는 "이 정도면 괜찮겠지"라는 이유로도 금지. 변조 사례 목록과 허용되는 리라이팅 범위는 `references/resume-refinement.md`의 "절대 금지 사항" 섹션 참조.

판단이 애매하면 **쓰지 않는다**. 갭은 갭으로 두고, 자소서에서 학습 의지·확장성으로 다룬다.

**v1.1 추가**: Step 5 리라이팅 사이클 종료 시 Fact-Checker 페르소나(Stage 0)가 원본 대조 Pass/Fail 판정. 1건이라도 Fail이면 해당 문장 원상 복구.

### 2. BCG 컨설턴트 관점의 회사 분석

회사 분석은 단순 회사 소개가 아니다. **재무 수치(매출 추세, 영업이익률, 현금흐름) + 산업 구조(Porter's 5 Forces) + 시장 전망(국내/글로벌) + 학자·이론 인용**으로 구성된 분석 보고서다. 검색 횟수는 10회를 기본으로 한다.

### 3. 멀티 페르소나 9명 평가 (v1.1 업데이트: 8 → 9)

문서 완성도는 9개 페르소나의 시각으로 평가한다:
- **Stage 0 (사실 검증)**: ⚫ FACT-CHECKER — 원본과의 1:1 대조 (Pass/Fail)
- **Stage 1 (5-Color)**: BLACK·RED·SILVER·BLUE·GOLD — 보편적 완성도
- **Stage 2 (채용 라인업)**: 헤드헌터·현업 매니저·HR — 실전 통과 가능성

Stage 1·2는 각 페르소나가 10점 만점으로 채점하고, 9점 미만 항목은 리라이팅 후 재평가. 최대 3회 반복. **Stage 0은 모든 리라이팅 직후 자동 실행되며, Fail 시 사이클 중단.**

### 4. 사용자 페르소나 존중 (v1.1 업데이트: 명시 확인 추가)

메모리·기존 자료에서 드러나는 사용자의 직무 정체성을 자소서 톤에 반영하되, **Step 4-0에서 1회 명시적으로 확인한 후 사용한다.** 자동 반영하지 않는다.

## 입력 점검

스킬 시작 시 다음 4가지가 모두 있는지 확인. 빠진 것은 사용자에게 요청한다.

1. **채용공고** (URL 또는 본문 텍스트)
2. **회사명** (공고에 있으면 추출, 모호하면 확인 요청)
3. **사용자 경력기술서** (텍스트 또는 업로드 파일)
4. **사용자 자기소개서 초안** (없으면 "초안 없이 새로 작성"으로 진행)

채용공고가 URL이면 `web_fetch`로 본문을 가져온다. 회사 홈페이지 URL도 사용자가 알려주면 함께 fetch한다.

## 워크플로우 (5단계 순차 실행)

각 단계는 자체 가이드 파일을 참조한다. 단계 시작 전에 해당 파일을 `view`로 읽는다.

### Step 1: 회사 분석 (BCG 컨설턴트 관점)
참조: `references/company-analysis.md`

웹검색 **10회 기본**. 다음을 수집한다:
- **재무·성장성**: 매출 추세 3~5년, 영업이익률, 현금흐름, 부채비율 (BCG 컨설턴트 관점)
- **산업 구조**: Porter's 5 Forces 5개 힘 평가
- **국내·글로벌 시장 전망**: 시장 규모, CAGR, 글로벌 포지셔닝
- **사업 포트폴리오**: BCG Matrix 또는 사업 라이프사이클 단계
- **지배구조·평판**: 오너 리스크, 잡플래닛/블라인드 시그널
- **JD 부서 정황**: 신설/보강, 직속 리더십

분석 본문에 적절한 학자·이론 인용 (Michael Porter, Christensen, Damodaran 등)을 1~2회 자연스럽게 포함.

### Step 2: JD ↔ 경력기술서 매칭 평가
참조: `references/jd-matching-rubric.md`

JD를 Must-have / Nice-to-have / Soft signals 세 층으로 분해하고, 각 항목에 0~3점 매핑. 종합 매칭률 산출.

### Step 3: 경력기술서 정교화
참조: `references/resume-refinement.md`

JD 키워드 미러링, 정량화, STAR 구조 압축, 순서 재배치. **거짓 사실 절대 추가 금지.** 변경 전후 비교표 작성.

### Step 4-0: 직무 정체성 확인 (v1.1 신규, 1회만)

Step 4 본격 진입 전에 자소서 톤의 기준이 될 직무 정체성을 사용자에게 명시적으로 확인한다.

**확인 절차**:

1. 메모리·이전 대화·제출 자료에서 직무 정체성 후보를 추출 (예: "Risk Solution Partner", "신뢰 기반 감사자")
2. 다음 형식으로 사용자에게 1줄 확인:

   > "자소서 전반에 일관되게 드러낼 본인의 직무 정체성/슬로건을 확인하겠습니다.
   > 감지된 정체성: **[추출된 후보]**
   > 그대로 사용 / 수정 / 새로 입력 중 선택해 주세요."

3. 후보가 추출되지 않으면 직접 질문:

   > "자소서 톤을 결정하기 위해 본인의 직무 정체성을 한 줄로 알려주세요.
   > 예: 'Risk Solution Partner', '데이터 기반 마케터', '현장 출신 운영 전문가' 등"

4. 사용자가 응답하면 그 정체성을 Step 4 전반에 일관되게 반영.

**왜 이 게이트가 필요한가**: 
- 메모리는 과거 시점 기록이고, 회사·직무별로 정체성을 미세조정할 필요가 있음
- 자동 반영은 사용자에게 "내가 모르는 결정이 일어났다"는 인상을 줌
- 1줄 확인 1회 = 토큰 비용 거의 없음, 톤 어긋남 사고 방지 효과 큼

### Step 4: 자기소개서 정교화
참조: `references/cover-letter-refinement.md`

회사 리서치 결과 + JD 매칭 결과 + Step 4-0 정체성을 결합. **전체 1000자 이내** (JD가 별도 지정하면 그것을 따름). 회사 고유 정보 1~2개, 정량 근거 1~2개, JD 키워드 미러링, 차별화 한 줄, 기여 방안 한 줄.

### Step 5: 멀티 페르소나 평가 및 리라이팅
참조: `references/persona-evaluation.md`

9개 페르소나(Stage 0 FACT-CHECKER + Stage 1 5색 + Stage 2 채용 라인업)로 경력기술서·자소서를 각각 평가하고, 9점 미만 항목을 리라이팅. 최대 3회 반복.

**v1.1 핵심 변경**: 매 리라이팅 직후 Stage 0 FACT-CHECKER가 자동 실행. Fail 시 해당 문장 원상 복구하고 사이클 종료.

평가 → 리라이팅 → 재평가 사이클이 끝나면 최종 .docx 2개를 갱신한다.

## Section 6 의사결정 매트릭스 가이드
참조: `references/decision-matrix.md`

Section 6의 종합 판단은 모호한 인상이 아니라 4축 가중평균(Growth 30% / Safety 30% / Meaning 25% / Fit 15%)으로 100점 산출. 각 축의 채점 기준과 근거 인용 방식은 위 참조 파일을 따른다.

## 분석 보고서 docx 생성 가이드
참조: `references/analysis-report.md`

분석 보고서는 다음 7개 섹션 + 표지로 구성:

1. 표지 (회사명·직무명·종합 점수·지원 권장도·분석 일자)
2. 회사 분석 (Section 1, BCG 컨설턴트 관점)
3. JD 매칭 대시보드 (Section 2)
4. 경력기술서 변경 전후 비교 (Section 3)
5. 자기소개서 변경 전후 비교 + 적용된 직무 정체성 (Section 4)
6. 멀티 페르소나 평가 결과 (Section 5, Stage 0 + Stage 1·2)
7. 종합 판단 및 권고 (Section 6, 의사결정 매트릭스)
8. 면접 예상 질문 Top 5 + 답변 키 포인트

채팅 마크다운에 이미 출력된 내용을 **그대로 docx로 옮긴다**. 재생성·요약하지 않아 토큰 부담 최소화.

## 진행 시 사용자에게 알리기

5단계가 다 끝나기까지 시간이 걸리므로, 시작 전 한 줄로 진행 계획을 알린다:

> "회사 분석 → 매칭 평가 → 이력서 정교화 → 직무 정체성 확인 → 자소서 정교화 → 9명 페르소나 평가(Fact-Checker 포함) → 3개 docx 생성(이력서·자소서·분석보고서) 순서로 진행하겠습니다. 웹검색 10회 + 평가 사이클 3회로 몇 분 걸립니다."

각 단계 완료 시 한 줄로 진행 상황 알림:
> "Step 1 완료. → Step 2 시작합니다."
> "Step 4-0: 직무 정체성 확인 필요. 1줄 응답 부탁드립니다."
> "Stage 0 FACT-CHECKER 검증 중..."
> "Stage 1 평가 진행 중 (1/3회)..."
> "분석 보고서 docx 생성 중... (회사 분석 + 매트릭스 + 면접 질문 포함, 약 1분)"
> "3개 docx 파일 모두 준비 완료."

장황한 중간 설명은 피한다.

## 트리거되지 않아야 하는 경우

- 단순 회사 정보 조회 ("○○회사 매출이 얼마야?")
- 채용공고나 본인 경력 자료 없이 일반론적 이직 조언만 요청
- 면접 예상 질문만 따로 요청 (별도 워크플로우)
- 연봉 협상, 처우 협의 문의

이런 경우엔 일반 응답으로 처리한다.
