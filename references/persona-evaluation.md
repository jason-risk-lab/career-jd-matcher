# 멀티 페르소나 평가 및 리라이팅 (Step 5) — v1.1

Step 3·4에서 정교화된 경력기술서와 자기소개서를 **3단계 평가 → 리라이팅 반복**을 통해 10점 만점에 가까운 완성도까지 끌어올린다.

**v1.1 변경**: Stage 0 FACT-CHECKER 페르소나 신규 추가. 매 리라이팅 직후 원본 대조 Pass/Fail 게이트 실행.

## 평가 철학

평가의 목적은 점수 자체가 아니라 **시각의 다양성**이다. 한 사람의 눈으로는 보이지 않는 결함을 9개 페르소나의 시각으로 잡아낸다. 페르소나마다 우선순위·기대치·민감 포인트가 다르기 때문에, 모두를 만족시킨 문서는 실제 채용 과정의 다양한 평가자에게도 통과할 가능성이 높다.

**v1.1 추가 원칙**: 점수가 높아도 사실 변조가 있으면 의미 없다. Stage 0이 그 안전망이다.

## 3단계 평가 구조

### Stage 0: FACT-CHECKER (사실 검증, Pass/Fail) — v1.1 신규

매 리라이팅 직후 자동 실행. 점수가 아닌 **Pass/Fail** 판정. Fail 시 해당 문장 원상 복구 + 사이클 중단.

### Stage 1: 5-Color Harness 평가 (보편적 완성도)

5-Color 방법론(BLACK·RED·SILVER·BLUE·GOLD)을 채용 문서 평가용으로 재해석. **문서 자체의 완성도**를 본다.

### Stage 2: 채용 의사결정자 라인업 평가 (실전 통과 가능성)

실제 채용 절차에서 문서를 보는 사람들의 시각. **실제 통과 가능성**을 본다.

각 단계(1·2) 종료 시 평균 점수와 항목별 점수를 산출하고, **9.0 미만이거나 단일 항목 7점 이하**가 있으면 리라이팅한 뒤 재평가한다. 최대 3회 반복.

## Stage 0: FACT-CHECKER (사실 검증 전담) — v1.1 신규

### ⚫ FACT-CHECKER — 원본 대조 (Pass/Fail 게이트)

**시각**: 리라이팅된 모든 문장을 사용자의 **원본 경력기술서/자소서와 1:1 대조**. 점수가 아닌 Pass/Fail 판정. 거짓 사실은 스킬 신뢰성의 핵심이므로 가장 먼저 검증.

**실행 시점**: 
- Step 3 (경력기술서 정교화) 직후
- Step 4 (자소서 정교화) 직후
- Stage 1·2 평가 후 리라이팅이 일어났을 때마다

**체크리스트 (항목별 Pass/Fail)**:

| # | 검증 항목 | 판정 기준 |
|---|----------|----------|
| 1 | 정량 수치 일치 | 모든 숫자(80건, 400건, 96% 등)가 원본에 존재하거나 원본 수치의 단위 변환에 한정 |
| 2 | 회사명·기간·직책 일치 | 재직 회사, 기간(YYYY.MM 단위), 직책이 원본과 100% 일치 |
| 3 | 프로젝트·자격증 일치 | 새로 등장한 프로젝트명·자격증·툴이 없음 |
| 4 | 단독/공동 구분 일치 | 팀 성과를 단독 성과로 둔갑시킨 곳 없음 ("TF 5명" → "본인 단독" 변환 금지) |
| 5 | 추정/예상 → 실제 둔갑 없음 | "약 X억 환수 예상" 같은 추정이 "X억 환수"로 변환되지 않음 |
| 6 | 인용 인물·기관 검증 | "○○ 대표가 직접 칭찬", "△△ 기관 표창" 등 검증 불가 인용이 새로 추가되지 않음 |
| 7 | 산업·도메인 경험 일치 | 가본 적 없는 산업 경험을 시사하지 않음 ("헬스케어 도메인 이해" 등) |
| 8 | 자격증·교육 일치 | CISA 진행 중을 "취득"으로 변환하지 않음 등 |

**판정 룰**:

- **8개 항목 모두 Pass** → 다음 단계 진행
- **1개라도 Fail** → 해당 문장 즉시 원상 복구 (또는 원본 표현으로 대체), 리라이팅 사이클 중단, 사용자에게 보고
- **모호한 경우 (예: 표현 강조가 사실 왜곡으로 보일 수 있음)** → 보수적으로 Fail 처리하고 사용자 판단 요청

**출력 형식**:

```markdown
### Stage 0: FACT-CHECKER 결과 ([회차]차)

| # | 검증 항목 | 판정 | 비고 |
|---|----------|------|------|
| 1 | 정량 수치 일치 | ✅ Pass | - |
| 2 | 회사명·기간·직책 일치 | ✅ Pass | - |
| 3 | 프로젝트·자격증 일치 | ❌ Fail | "FCPA 운영 경험" 문장이 원본에 없음 → 해당 문장 삭제 |
| 4 | 단독/공동 구분 일치 | ✅ Pass | - |
| 5 | 추정/예상 → 실제 둔갑 | ✅ Pass | - |
| 6 | 인용 인물·기관 검증 | ✅ Pass | - |
| 7 | 산업·도메인 경험 일치 | ✅ Pass | - |
| 8 | 자격증·교육 일치 | ✅ Pass | - |
| **종합** | - | **❌ FAIL** | 1건 수정 적용 후 사이클 종료 |
```

**왜 이 페르소나가 필요한가**:

- Stage 1·2의 8개 페르소나는 "잘 썼는가"를 평가하다 보니, 점수를 끌어올리려는 리라이팅에서 미세한 거짓이 끼어들 수 있음
- 3회차쯤 되면 누적 변형으로 원본과 미세하게 어긋난 문장이 만들어질 위험
- Pass/Fail 게이트를 마지막에 두면 "거짓 금지" 원칙이 평가 시스템 안에 구조적으로 내장됨
- 사용자가 신뢰 기반 직무 정체성(예: "Risk Solution Partner")을 가진 경우 특히 critical

## Stage 1: 5-Color Harness (5 페르소나)

### 🟤 BLACK — 수행자 (형식·구조·가독성)

**시각**: 문서를 처음 손에 받아 든 사람의 첫 인상. 오탈자·문법·서식·구조 정합성을 본다. 본문 내용보다 "이 사람이 문서를 만드는 기본기를 갖췄는가"를 판단.

**평가 기준 (각 1점, 합 10점)**:
- (1) 오탈자·문법 오류 없음
- (2) 종결 체 일관 (자소서는 `~습니다` 체, 경력기술서는 명사형/`~함` 체)
- (3) 들여쓰기·불릿 레벨 일관
- (4) 정량 수치 표기 일관 (200만원 / 200만 원 / 2,000,000원 통일)
- (5) 회사명·고유명사 표기 일관 (루닛/Lunit, SPC/㈜파리크라상)
- (6) 날짜 표기 일관 (2025.04 / 2025년 4월)
- (7) 영문 약자 적절 (CISA, CIA 같은 약어 첫 등장 시 풀어쓰기)
- (8) 섹션 길이 균형 (한 섹션만 비대하지 않음)
- (9) 시각적 위계 명확 (헤더-본문-불릿 위계가 한눈에)
- (10) 전체 분량 적정 (자소서 1000자 ±10%, 경력기술서는 직무 경력에 비례)

### 🔴 RED — 이성 (논리·근거·정량성)

**시각**: 주장과 근거의 연결을 본다. "그래서?" "왜?"를 끊임없이 묻는 분석가.

**평가 기준 (각 1점, 합 10점)**:
- (1) 모든 주장에 근거(숫자·사례·기간)가 따르는가
- (2) 정량 수치가 충분히 등장하는가 (자소서 1~2개+, 경력기술서 항목당 1개+)
- (3) 인과 관계가 명확한가 ("A를 했더니 B 결과가 나왔다")
- (4) 논리적 비약 없음 (작은 경험을 거대 성과로 부풀리지 않음)
- (5) STAR 구조(상황-과제-행동-결과)가 주요 프로젝트에 적용되어 있는가
- (6) 시간 순서·중요도 순서가 일관된가
- (7) 자기 평가어("저는 우수합니다")보다 사실 기반("400건 처리, 96% 해결률")
- (8) 모호한 형용사("다양한·많은·여러") 최소화
- (9) JD 요구사항과의 매핑이 1대1로 추적 가능한가
- (10) 갭(약점)을 솔직히 인정하는 곳이 있는가 (역으로 신뢰도 ↑)

### ⚪ SILVER — 분야 전문가 (직무 깊이·전문성)

**시각**: 같은 직무 시니어가 읽는다. 전문 용어 정확성, 깊이 있는 통찰, 업계 상식 부합도를 본다. 어설픈 표현을 첫눈에 잡아낸다.

**평가 기준 (각 1점, 합 10점)**:
- (1) 전문 용어를 정확히 사용 (예: "내부감사" vs "감사" vs "회계감사" 구분)
- (2) JD 산업의 핵심 개념 이해 (의료 AI라면 SaMD·CDx·FDA·식약처 등)
- (3) 직무 표준 프레임워크 언급 (COSO ERM, IIA Standards, ISO 37301 등)
- (4) 정량 성과가 직무 상식 범위 내 (감사 80건/5년 = 신뢰 가능 수준)
- (5) 깊이 있는 통찰 ("왜 그렇게 했는가"의 사고 프로세스)
- (6) 업계 트렌드 인지 (ESG 의무화, 디지털 감사, AI 활용 등)
- (7) 실무 흐름이 자연스러움 (피상적 키워드 나열이 아닌)
- (8) 시니어가 보기에 "이 사람과 일할 만하다"는 신호
- (9) 본인 한계의 솔직한 인정 (모르는 영역을 안다고 우기지 않음)
- (10) 차별화 포인트가 분명 (다른 지원자와 구별되는 본인만의 강점)

### 🔵 BLUE — 공감 (톤·진정성·인간미)

**시각**: 이 사람과 함께 일하고 싶은가. 차가운 스펙 나열인가, 사람 냄새가 나는가.

**평가 기준 (각 1점, 합 10점)**:
- (1) 진정성 있는 동기 (회사 이름만 바꾸면 다른 자소서에도 통할 일반론이 아님)
- (2) 직무·산업에 대한 진짜 관심이 드러남
- (3) 본인 신념·가치관이 자연스럽게 노출 (선언적이지 않게)
- (4) 과장 없음 ("최고의·탁월한·완벽한" 자기 평가어 절제)
- (5) 약점 인정 시 비굴하지 않고 담담함
- (6) 본인 정체성의 일관성 (자기소개 헤더·본문·결론의 톤 일치)
- (7) 회사·동료에 대한 존중 톤
- (8) "함께 성장하고 싶다"가 일방적 충성 맹세가 아니라 상호적 관계로
- (9) 감정이 절제되어 있으면서도 따뜻함
- (10) 글에서 사람이 보이는 정도 (페르소나가 생생함)

### 🟡 GOLD — 독자 (최종 채용 결정자의 본능)

**시각**: 모든 분석을 끝낸 의사결정자가 마지막으로 "그래서 이 사람 뽑을까?"를 본능적으로 판단하는 순간. 직관·인상·기억에 남는 한 줄.

**평가 기준 (각 1점, 합 10점)**:
- (1) 첫 3줄에서 "더 읽고 싶다"가 생기는가
- (2) 한 문장으로 요약 가능한 차별화 포인트가 있는가
- (3) 끝까지 읽고 나서 인상에 남는 키워드가 있는가
- (4) "이 사람을 면접에서 만나보고 싶다"는 본능이 작동하는가
- (5) 자소서를 다 읽고 나서 회사·직무와의 적합성이 정렬되어 느껴지는가
- (6) 약점이 결격사유로 안 느껴지는가 (또는 약점을 충분히 상쇄하는가)
- (7) 다른 후보 100명 중 기억할 만한 요소가 있는가
- (8) 톤이 자신감 있으면서도 거만하지 않은가
- (9) 회사 관점에서 "리스크 없는 안전한 채용"으로 느껴지는가
- (10) 면접 후 동료에게 "한번 만나봐"라고 추천할 정도의 흥미를 유발하는가

## Stage 2: 채용 의사결정자 라인업 (3 페르소나)

### 🟦 헤드헌터/리크루터

**시각**: 매일 수십~수백 장의 이력서를 보는 사람. 평균 검토 시간 6~30초. ATS 키워드 매칭과 한눈에 들어오는 깔끔함이 핵심.

**평가 기준 (각 1점, 합 10점)**:
- (1) JD 핵심 키워드 5개 이상이 경력기술서·자소서에 명시
- (2) 경력 연차·직급이 한눈에 들어옴
- (3) 최근 3개 회사·재직기간이 페이지 상단에서 보임
- (4) 정량 성과가 첫 페이지에서 잡힘
- (5) 자격증·기술 스택이 별도 섹션으로 정리
- (6) 짧고 강한 한 줄 자기소개 ("Risk Solution Partner" 같은)
- (7) PDF 변환 시 깨질 위험이 없는 깔끔한 서식
- (8) 이메일·연락처 정확
- (9) 직무명·회사명에 검색 가능한 키워드 사용
- (10) 6초 안에 "JD에 맞는 후보"임이 판단 가능

### 🟥 현업 채용 매니저 (직속 상사 페르소나)

**시각**: 이 사람과 매일 같이 일할 사람. 실무 즉시 투입 가능성, 직무 깊이, 협업 가능성을 본다.

**평가 기준 (각 1점, 합 10점)**:
- (1) 직무 핵심 역량(필수요건)이 충족됨
- (2) 즉시 투입 가능한 실무 수준 (단순 이론이 아닌)
- (3) 본인이 한 일과 팀이 한 일이 구분되어 있음
- (4) 본인의 의사결정·판단 사례가 보임
- (5) 어려운 상황을 어떻게 풀었는지 구체적 사례
- (6) 회사 산업·맥락에 대한 이해도
- (7) 본인 한계를 어떻게 보완하려는지 (학습 의지)
- (8) 협업 스타일 (단독 플레이어 vs 팀 플레이어 균형)
- (9) "이 사람이 입사하면 6개월 안에 임팩트를 낼 수 있겠다"는 신호
- (10) 만났을 때 깊이 있는 대화가 가능할 것 같은 흔적

### 🟨 HR/People Team

**시각**: 컬쳐핏, 커뮤니케이션, 장기 근속 가능성을 본다. 단순 스펙보다 "이 사람이 우리 조직에 녹아들 수 있는가"를 판단.

**평가 기준 (각 1점, 합 10점)**:
- (1) 회사 가치관·문화에 대한 이해와 정렬
- (2) 자기소개서의 문장력 (회사 내부 커뮤니케이션 가능성 추정)
- (3) 이전 회사 퇴사 사유의 합리성 (책임 전가가 아닌 성숙한 인정)
- (4) 협업·갈등 관리 사례
- (5) 장기 비전이 회사 미션과 정렬되어 있음
- (6) 너무 짧은 재직기간이 반복되지 않음 (또는 합리적 설명)
- (7) 이직 동기의 진정성 (단순 연봉·조건 상승이 아닌)
- (8) 자기 성찰·성장 의지가 보임
- (9) 다양성·포용성에 열린 태도
- (10) 면접에서 만나도 인격적 결격 사유가 없을 것 같음

## 평가 → 리라이팅 사이클 (v1.1 업데이트)

### 사이클 전체 흐름

```
[Step 3·4 정교화 산출물]
    ↓
[Stage 0 FACT-CHECKER] ─── Fail ──→ 원상 복구 + 사이클 중단
    ↓ Pass
[Stage 1 평가] + [Stage 2 평가] (1회차)
    ↓
[리라이팅] (7점 이하 항목)
    ↓
[Stage 0 FACT-CHECKER 재실행] ─── Fail ──→ 원상 복구
    ↓ Pass
[Stage 1·2 재평가] (2회차)
    ↓
... (3회차까지 반복)
    ↓
[최종 산출]
```

### 1회차 평가

각 페르소나가 10점 만점으로 채점. 항목별 점수 + 핵심 코멘트. **Stage 0은 그 직전에 Pass/Fail 출력.**

**출력 표 (Stage 0)**:
```markdown
### Stage 0 FACT-CHECKER (1회차)
[위 Stage 0 섹션의 8항목 체크리스트 표]
```

**출력 표 (Stage 1)**:

```markdown
| 페르소나 | 점수 | 강점 | 약점 (개선 필요) |
|----------|------|------|-----------------|
| 🟤 BLACK | X/10 | ... | ... |
| 🔴 RED | X/10 | ... | ... |
| ⚪ SILVER | X/10 | ... | ... |
| 🔵 BLUE | X/10 | ... | ... |
| 🟡 GOLD | X/10 | ... | ... |
| **평균** | **X.X/10** | - | - |
```

**Stage 2도 동일 형식.**

### 2회차 평가 (리라이팅 후)

1회차에서 7점 이하 항목을 식별하고, 해당 항목을 개선하는 방향으로 본문을 리라이팅. 단 **여전히 거짓 사실은 금지**. 리라이팅 직후 Stage 0 재실행. 재평가하여 점수 변화를 본다.

### 3회차 평가 (최종)

마지막 보정. 평균 9.0 이상, 단일 항목 7점 이하가 없는 상태를 목표로. Stage 0은 마지막에 한 번 더 실행.

### 종료 조건

다음 중 하나를 만족하면 종료:

1. **모든 페르소나 평균 9.0 이상 + 단일 항목 7점 이하 없음 + Stage 0 Pass**
2. **3회 반복 완료** (점수가 더 이상 의미 있게 오르지 않으면 8점대 후반에서 종료해도 무방, 단 Stage 0 Pass 필수)
3. **Stage 0 Fail로 강제 중단** — 해당 시점의 사실 보존 산출물을 최종으로 사용

3회 반복에도 10점 만점이 안 나오면 그것은 자료의 한계(경험·근거 부족)일 가능성이 크다. 거짓을 추가하지 말고 솔직히 "이 항목은 본인의 실제 경험상 X점이 한계"라고 보고한다.

## 거짓 사실 추가 절대 금지 (재강조)

리라이팅 과정에서 점수를 올리려고 다음을 하면 안 된다:

- 없는 프로젝트·경험·자격증 추가
- 정량 수치 부풀리기 (실제 80건을 100건으로)
- 팀 성과를 본인 단독 성과로 표시
- 직책·재직기간 변조
- 만난 적 없는 사람·기관 인용

리라이팅은 **표현·구조·강조점·매핑**의 조정에 한정한다. 사실은 사실로 둔다. 만약 평가 페르소나가 "이 부분에 정량 수치가 필요하다"고 지적했는데 본인이 가진 수치가 없다면, 거짓 수치를 만들지 말고 그 항목 점수를 그대로 둔다.

**v1.1 추가**: Stage 0 FACT-CHECKER가 이 원칙의 자동 게이트 역할을 한다. 사람이 깜빡해도 시스템이 잡는다.
